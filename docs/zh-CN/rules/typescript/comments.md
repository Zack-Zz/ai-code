# TypeScript/JavaScript 注释与 JSDoc

> 本文档扩展 [common/coding-style.md](../common/coding-style.md) 的 TypeScript/JavaScript 注释规范。

## 通用规则

- 注释应解释意图、约束、权衡或非显而易见行为
- 不要逐行复述代码
- 过时注释应删除，不要保留误导信息
- 注释尽量短小并贴近代码，确有必要再写较长说明

## JSDoc / TSDoc

- 导出函数、共享 hook、公有类、复杂公有类型需要 JSDoc
- 说明输入约束、返回值和重要副作用
- 调用方需要显式处理时，应记录可能抛出的错误
- 示例应简洁并贴近真实用法

## 必须写注释的场景

- 复杂解析逻辑
- 不直观正则表达式
- 安全敏感分支与校验假设
- 框架/工具链限制下的 workaround
- 性能敏感且实现选择不明显的代码

## TODO / FIXME 规则

- `TODO`、`FIXME` 必须包含可执行上下文
- 优先附 issue、任务 ID 或明确触发条件
- 已完成事项应及时移除 TODO/FIXME

## 不推荐模式

- 噪声注释（如“变量加一”“赋值”）
- 无实际价值的大段横幅注释
- 生产文件中保留注释掉的代码
- 没有负责人或后续条件的“临时”注释
